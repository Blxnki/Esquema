Funciones

Una función es un conjunto de sentencias que realizan una tarea específica, las funciones permiten dividir un programa en partes más manejables y facilitan la reutilización de código.
Se declaran con la palabra clave function, seguida del nombre de la función y paréntesis para los argumentos.
El cuerpo de la función se escribe entre llaves {}.
Las funciones pueden recibir argumentos y devolver valores mediante la sentencia return.
En JavaScript, las declaraciones de funciones se mueven al principio del programa.
Las funciones pueden ser llamadas antes de ser definidas en el código.
Se pueden usar como valores, almacenarse en variables o pasarse como argumentos a otras funciones.

INVOCACIÓN DE UNA FUNCIÓN

Las funciones pueden tener argumentos con valores por defecto, estos valores se utilizan cuando no se proporciona un argumento al llamar la función.
function calcularImporte(importeNeto, iva = ivaGeneral)
Se pueden crear funciones que acepten un número indeterminado de argumentos.
function nombreFuncion(...listaArgumentos)
Los argumentos se pueden recorrer usando un bucle for...of.
En JavaScript, los argumentos se pasan por valor a las funciones.
Esto significa que se hace una copia del valor del argumento.
Las modificaciones a los argumentos dentro de la función no afectan a las variables originales fuera de la función.

EL TIPO DE DATOS FUNCTION

En JavaScript, las funciones son objetos de tipo Function.
Esto se puede comprobar usando el operador typeof en una función, que devolverá "function".
Las funciones pueden ser almacenadas en variables, pasadas como argumentos a otras funciones y devolver valores.
Las funciones pueden ser utilizadas como valores en expresiones y pueden ser almacenadas en arrays y objetos.
Pueden declararse como funciones y tambien como anonimas

ALCANCE DE LAS VARIABLES

Variables globales: Se definen en el bloque principal del código y son accesibles desde cualquier parte del programa, incluyendo dentro de funciones.
Variables locales: Se declaran dentro de una función o bloque de código y solo son accesibles dentro de ese ámbito específico.
Las variables globales pueden ser leídas y modificadas tanto dentro como fuera de las funciones.
Las variables locales son inaccesibles fuera de su ámbito y se destruyen al finalizar la ejecución de la función o bloque.
Es posible tener variables globales y locales con el mismo nombre sin que esto cause un error, ya que pertenecen a ámbitos diferentes.
Dentro de una función, JavaScript da preferencia a las variables locales sobre las globales con el mismo nombre.
