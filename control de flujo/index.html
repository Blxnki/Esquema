SENTENCIAS DE CONTROL DE FLUJO

if..else

La sentencia condicional if...else en JavaScript permite ejecutar bloques de código diferentes según se cumpla o no una condición. Su sintaxis básica es la siguiente:
if (condición) {
                    // Código a ejecutar si la condición es verdadera
                  } else {
                    // Código a ejecutar si la condición es falsa
                  }
Las llaves {} son opcionales si el bloque contiene una sola sentencia.
La cláusula else es opcional, permitiendo una versión simplificada:
if (condición) {
                    // Código a ejecutar si la condición es verdadera
                  }
Se pueden anidar múltiples condiciones usando else if:
if (condición1) {
                    // Código a ejecutar si la condición1 es verdadera
                    } else if (condición2) {
                        // Código a ejecutar si la condición2 es verdadera
                    } else {
                        // Código a ejecutar si ninguna condición es verdadera
                            }
Es importante notar que en sentencias if anidadas, cada condición se evalúa solo si las anteriores son falsas. Esto permite crear lógicas de decisión complejas y flexibles.
Para condiciones más complejas con múltiples casos, puede ser más apropiado usar la sentencia switch...case, especialmente cuando se compara una variable contra valores específicos.

switch..case

La sentencia switch en JavaScript permite ejecutar bloques de código basados en el valor de una expresión, siendo una alternativa más legible a múltiples condiciones if...else if. Su sintaxis es la siguiente:
switch (expresión) {
                    case valor1:
                      // Código a ejecutar si expresión === valor1
                      break;
                    case valor2:
                      // Código a ejecutar si expresión === valor2
                      break;
                    default:
                      // Código a ejecutar si no hay coincidencia
                  }
Funcionamiento:
Se evalúa el valor de la expresión proporcionada.
Se compara el valor de la expresión con los valores especificados en cada case usando igualdad estricta (===).
Si hay coincidencia, se ejecuta el bloque de código correspondiente.
La sentencia break detiene la ejecución del switch. Si se omite, los bloques de los casos siguientes también se ejecutarán ("fall-through").
Si no hay coincidencias, se ejecuta el bloque asociado a default, que es opcional.

Aunque es útil para comparar valores específicos, no es adecuado para condiciones complejas o rangos.

for

El bucle for en JavaScript es una estructura de control que permite ejecutar un bloque de código un número determinado de veces. Su sintaxis es la siguiente:
for (inicialización; condición; expresión-final) {
                    // Código a ejecutar en cada iteración
                  }
Componentes del bucle for:
Inicialización: Se ejecuta una vez al inicio del bucle, generalmente para declarar e inicializar una variable contadora.
Condición: Evalúa si el bucle debe continuar. Si es true, se ejecuta el bloque de código; si es false, el bucle termina.
Expresión final: Se ejecuta al final de cada iteración, generalmente para actualizar o incrementar la variable contadora.
Opcionalidad: Las tres expresiones (inicialización, condición, expresión-final) son opcionales:
Si falta la inicialización, la variable debe declararse antes del bucle.
let x = 1;
                    for (; x = 10; x++) {
                      console.log(x);
                    }
Si falta la condición, se evalúa siempre como true, creando un bucle infinito a menos que se use break.
for (let i = 1;; i++) {
                    if (i > 10) break;
                    console.log(i);
                  }
Si falta la expresión final, la actualización debe realizarse dentro del bloque del bucle.
for (let i = 1; i = 10;) {
                    console.log(i);
                    i++;
                  }
Eficiencia: El bucle for es más compacto y menos propenso a errores que otros bucles como while.
Flexibilidad: Puede adaptarse a diferentes escenarios, como recorrer arrays o realizar cálculos repetitivos

While

El bucle while en JavaScript es una estructura de control que ejecuta repetidamente un bloque de código mientras se cumpla una condición específica. Su sintaxis es:
while (condición) {
                    // bloque de código
                  }
Funcionamiento:
Se evalúa la condición.
Si es true, se ejecuta el bloque de código.
Se vuelve a evaluar la condición.
Si sigue siendo true, se repite el proceso. Si es false, se sale del bucle.

do..while

La sentencia do...while es similar, pero garantiza que el bloque de código se ejecute al menos una vez:
do {
                    // bloque de código
                  } while (condición);
La principal diferencia es que do...while siempre ejecuta el bloque al menos una vez, incluso si la condición es falsa inicialmente.

Break y continue

La sentencia break y continue son herramientas importantes para controlar el flujo de ejecución en bucles. Veamos sus diferencias y usos:
Break:
Termina inmediatamente la ejecución del bucle.
Uso en switch: Se usa al final de cada case para evitar la ejecución de los casos siguientes.
Uso en bucles: Permite salir completamente del bucle antes de que se cumpla la condición de finalización normal.
Continue:
Salta el resto del código en la iteración actual y pasa a la siguiente iteración del bucle.
Útil cuando quieres omitir parte del código del bucle en ciertas condiciones, pero continuar con las siguientes iteraciones.
