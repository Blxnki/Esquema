Operadores

Los operadores se agrupan en las siguientes categorías:
Aritméticos. Realizan operaciones matemáticas sobre sus operandos.
Asignación. Asignan el valor del operando derecho al izquierdo. El más conocido es ʻ=ʼ.
Comparación. Tal como indica su nombre, los operadores de este grupo comparan los operandos y devuelven el valor true si son iguales o false en caso contrario. Dichos operandos podrán ser numéricos, lógicos, cadenas de caracteres e, incluso, objetos.
Al igual que los operadores de la categoría anterior, estos también devuelven el valor true o false, pero a diferencia de ellos, dicho valor se obtiene aplicando las reglas de algebra de Boole.
condición?valor1:valor2. Operador ternario formado por una condición y dos valores. La condición es una expresión de comparación y/o lógica cuyo resultado (true o false) determina si el resultado devuelto es el primer o el segundo valor.
typeof operando. Operador unario que devuelve el tipo de datos al que pertenece su operando.

OPERADORES ARITMÉTICOS

Operadores binarios:
Suma (+): suma números o concatena cadenas
Resta (-)
Multiplicación (*)
División (/)
Resto (%)
Exponenciación (**)
Incremento (++)
Decremento (--)
Negación (-)
Características importantes:
El operador '+' es especial: suma números y concatena cadenas. Por ejemplo: 5 + 5 = 10, pero "5" + 5 = "55".
Conversión automática: si un operando es cadena, el otro se convierte a cadena en la suma. Por ejemplo: 5 + "5" = "55".
Con otros operadores, las cadenas numéricas se convierten a números.
Incremento y decremento pueden ir antes o después de la variable, afectando cuándo se aplica el cambio.

Asignación simple (=)
Asignación aditiva (+=):Asigna a la variable situada a la izquierda el resultado de sumar su valor inicial al de la expresión derecha.
Asignación multiplicativa (*=)
Asignación de resto (%=)
Asignación de exponente (**=)
Asignación de división (/=)
Asignación de resta(-=)
El operador += también se usa para concatenar cadenas. Si uno de los operandos es una cadena, JavaScript convierte automáticamente el otro en una cadena y los concatena.
Es posible usar múltiples operadores de asignación en una misma línea.
Estos operadores de asignación compuestos proporcionan una forma concisa de realizar operaciones y asignaciones en una sola expresión, lo que puede hacer que el código sea más eficiente y legible.

OPERADORES DE COMPARACIÓN

Los operadores de comparación en JavaScript son herramientas fundamentales para evaluar condiciones y controlar el flujo de ejecución del programa. Estos operadores devuelven un valor booleano (true o false) basado en la comparación de sus operandos. JavaScript ofrece los siguientes operadores de comparación:
Igualdad (==): Compara si los valores de los operandos son iguales, realizando conversiones de tipo si es necesario.
Igualdad estricta (===): Compara si los valores y tipos de los operandos son idénticos, sin realizar conversiones.
Desigualdad (!=): Compara si los valores de los operandos son diferentes, realizando conversiones de tipo si es necesario.
Desigualdad estricta (!==): Compara si los valores o tipos de los operandos son diferentes, sin realizar conversiones.
Menor que , menor o igual que, mayor que , mayor o igual que: Comparan valores numéricos o lexicográficamente las cadenas.
Características importantes:
Las comparaciones de cadenas se realizan utilizando el orden lexicográfico basado en los valores Unicode de los caracteres
Al comparar una cadena con un número, JavaScript intenta convertir la cadena a un número.
Una cadena vacía ("") se convierte en 0 para comparaciones numéricas.
Se recomienda usar comparaciones estrictas (=== y !==) para evitar conversiones de tipo inesperadas.
null y undefined son considerados iguales con ==, pero diferentes con ===.

OPERADORES LÓGICOS

Los operadores lógicos en JavaScript aplican las reglas del álgebra de Boole y se utilizan solos o en combinación con operadores de comparación. JavaScript ofrece tres operadores lógicos principales:
AND(&&):
Devuelve true si ambos operandos son true, false en caso contrario.
Para valores no booleanos, devuelve el valor del primer operando si este se puede convertir a false. De lo contrario, devuelve el valor del segundo operando.
OR(||)
Devuelve true si alguno de sus operandos es true, false en caso contrario.
Para valores no booleanos, devuelve el valor del primer operando si este se puede convertir a true. De lo contrario, devuelve el valor del segundo operando.
NOT (!)
Devuelve el valor opuesto del operando, true si es false y viceversa.
Para valores no booleanos, devuelve false si su único operando se puede convertir a true. De lo contrario, devuelve true.
JavaScript convierte a false los siguientes valores en operaciones lógicas:
undefined, null, NaN, cadena vacía y número 0

OPERADORES condicional

Los operadores condicionales en JavaScript permiten evaluar una condición y ejecutar una acción basada en el resultado de esa evaluación:
condición ? valor1 : valor2
Si la condición es true, devuelve valor1.
Si la condición es false, devuelve valor2.
Este operador es útil para simplificar estructuras condicionales (if-else) en una sola línea.

Operador typeof

El operador typeof devuelve el tipo de dato de un valor:
typeof valor
Además de los tipos de datos primitivos (String, Number, BigInt, Boolean, Null y Undefined), typeof también identifica dos tipos adicionales:
Object: Representa estructuras de datos formadas por atributos y métodos, fundamentales en la programación orientada a objetos.
Function: Aunque poco común, las funciones son consideradas un tipo de dato especial en JavaScript, siendo el único tipo invocable.
typeof null // Devuelve "object"
Esta respuesta es considerada una inconsistencia en el lenguaje. Aunque null es un valor primitivo, typeof null devuelve "object". Esto se debe a un error histórico en la implementación original de JavaScript que se ha mantenido por razones de compatibilidad. Algunos desarrolladores lo consideran un error, mientras que otros lo justifican argumentando que null se usa frecuentemente para indicar una referencia vacía a un objeto.
Esta peculiaridad de typeof null es un aspecto importante a tener en cuenta al trabajar con tipos de datos en JavaScript, especialmente cuando se realizan comprobaciones de tipo.

PRECEDENCIA DE OPERADORES

Orden de ejecución: No siempre es de izquierda a derecha o de derecha a izquierda. La precedencia de los operadores determina el orden.
Precedencia de operadores (de mayor a menor):
ncremento/decremento (++, --)
Negación y typeof (-, typeof)
Exponenciación (**)
Multiplicación, división, resto (*, /, %)
Adición y sustracción (+, -)
Comparación
Igualdad (==, !=, ===, !==)
Operador condicional (?:)
Asignación (=, +=, -=, *=, /=, %=, **=)
Uso de paréntesis: Se pueden usar para modificar el orden de ejecución. Por ejemplo, (2 + 3) * 5 ** 2 daría como resultado 625.
Paréntesis como operador de agrupación: Tienen la mayor precedencia y pueden usarse para forzar un orden de evaluación específico.
